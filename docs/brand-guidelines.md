# Workout Planner Brand Guidelines

This document defines the visual identity and brand guidelines for the Workout Planner application ecosystem.

## Brand Identity

Workout Planner is a comprehensive fitness coaching platform that combines AI-powered personalization with evidence-based training principles. The brand identity reflects professionalism, energy, and technological innovation.

## Color Palette

### Primary Colors

**Athletic Blue (#2196F3)**
- Main brand color
- Use for: Primary buttons, app bars, key actions, brand elements
- Represents: Trust, professionalism, technology
- Full scale available: 50-900 for different states (hover, pressed, disabled)

**Energy Orange (#FF9800)**
- Secondary brand color
- Use for: Accents, secondary actions, highlights, call-to-action elements
- Represents: Energy, motivation, achievement
- Full scale available: 50-900

### Semantic Colors

**Success Green (#4CAF50)**
- Use for: Completed workouts, goal achievements, positive feedback
- Conveys: Progress, health, accomplishment

**Warning Yellow (#FFC107)**
- Use for: Alerts, reminders, important notices
- Conveys: Attention, caution, important information

**Error Red (#F44336)**
- Use for: Errors, critical issues, deletion actions
- Conveys: Urgency, problems, warnings

**Info Blue (#03A9F4)**
- Use for: Informational messages, tips, educational content
- Conveys: Knowledge, guidance, support

### Neutral Colors

**Grayscale Palette**
- Use for: Text, backgrounds, borders, dividers
- White (#FFFFFF): Primary background (light mode)
- Black (#000000): Primary background (dark mode)
- Gray 900 (#212121): Primary text color
- Gray 600 (#757575): Secondary text color
- Gray 400 (#BDBDBD): Disabled text, placeholders
- Gray 200 (#EEEEEE): Borders, dividers
- Gray 100 (#F5F5F5): Secondary backgrounds

## Typography

### Font Family

**System Default**
- Uses platform-native fonts for optimal readability and performance
- iOS: San Francisco
- Android: Roboto
- Web: -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, sans-serif

### Type Scale

Based on Material Design 3 type system:

**Display** (Large headlines, hero text)
- Large: 57px / 64px line height
- Medium: 45px / 52px line height
- Small: 36px / 44px line height

**Headline** (Section headers)
- Large: 32px / 40px line height
- Medium: 28px / 36px line height
- Small: 24px / 32px line height

**Title** (Card titles, list headers)
- Large: 22px / 28px line height
- Medium: 16px / 24px line height (Medium weight)
- Small: 14px / 20px line height (Medium weight)

**Body** (Primary content)
- Large: 16px / 24px line height
- Medium: 14px / 20px line height
- Small: 12px / 16px line height

**Label** (Buttons, tabs, chips)
- Large: 14px / 20px line height (Medium weight)
- Medium: 12px / 16px line height (Medium weight)
- Small: 11px / 16px line height (Medium weight)

### Font Weights

- Regular (400): Body text, descriptions
- Medium (500): Labels, buttons, emphasized text
- Semibold (600): Subheadings (optional)
- Bold (700): Strong emphasis (use sparingly)

## Spacing & Layout

### Base Unit

4px base unit ensures consistent spacing across all platforms.

### Spacing Scale

- XS (4px): Tight spacing, icon padding
- SM (8px): Component internal spacing
- MD (12px): Compact cards, dense layouts
- Base (16px): Default padding, standard gaps
- LG (24px): Section spacing, comfortable padding
- XL (32px): Screen margins, large gaps
- 2XL (48px): Major section spacing
- 3XL (64px): Hero section spacing

### Border Radius

- SM (4px): Chips, small badges
- MD (8px): Buttons, input fields
- LG (12px): Cards, modals
- XL (16px): Large cards, bottom sheets
- Full (9999px): Pills, circular avatars

### Icon Sizes

- XS (16px): Inline icons, dense lists
- SM (20px): List item icons
- MD (24px): Standard app bar icons
- LG (32px): Feature icons
- XL (48px): Hero icons, large avatars

## Logo Usage

### Primary Logo

- Use on white or light backgrounds
- Maintain clear space around logo (minimum spacing = logo height × 0.5)
- Never distort or stretch the logo
- Minimum size: 120px width for digital, 1 inch for print

### Logo Variations

- **Full Color**: Primary use case
- **White**: Use on dark backgrounds
- **Single Color**: Use when color is limited (Athletic Blue preferred)
- **Icon Only**: Use when space is constrained (minimum 32px)

### Logo Don'ts

- Don't change colors
- Don't add effects (shadows, gradients, outlines)
- Don't rotate or skew
- Don't place on busy backgrounds without sufficient contrast
- Don't use old logo versions

## Visual Style

### Photography

- Active, energetic shots showing real people
- Natural lighting preferred
- Focus on achievement and progress
- Diverse representation of fitness levels and demographics

### Illustrations

- Clean, modern style
- Use brand color palette
- Flat or subtle gradients
- Avoid overly detailed or realistic illustrations

### Icons

- Material Design icon set
- 24dp standard size
- Line weight: 2dp
- Rounded corners preferred
- Fill icons for active states, outline for inactive

## Accessibility

### Color Contrast

All text must meet WCAG AA standards:
- Normal text: Minimum 4.5:1 contrast ratio
- Large text (18px+): Minimum 3:1 contrast ratio
- UI components: Minimum 3:1 contrast ratio

### Text Sizing

- Minimum body text size: 14px (mobile), 16px (desktop)
- All text must be scalable
- Support for system font size preferences

### Interactive Elements

- Minimum touch target: 44×44px (mobile)
- Clear focus indicators for keyboard navigation
- Sufficient spacing between interactive elements

## Platform-Specific Guidelines

### iOS

- Follow iOS Human Interface Guidelines
- Use native navigation patterns
- Respect system font sizing
- Support Dark Mode

### Android

- Follow Material Design guidelines
- Use native navigation patterns (bottom nav, drawer)
- Support system theming
- Respect system font scaling

### Web

- Responsive design (mobile-first)
- Support browser zoom (up to 200%)
- Keyboard navigation support
- Cross-browser compatibility
